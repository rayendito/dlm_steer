TIMPA RUN SEMANTICS REPORT
==========================

This report follows the current run.py behavior. It uses two real examples from
each dataset's train split (currently val.csv). Numeric sampling arguments such
as temperature, margin, alpha, and refill steps are intentionally omitted.

Important conventions
---------------------

1. IMDb and catdog are paired datasets. Each source side is rewritten toward
   the other side, so each method runs in both directions.
2. ELIFive and Dolly have one base text collection. That same collection is
   reused for each target prompt. Only two of the three targets are illustrated
   below; the third target (phd for ELIFive and flirty for Dolly) also runs.
3. The activation-vector extraction corpora are the one-word concept labels,
   not the dataset sentences. For example, steering toward negative uses
   corpus1=["negative"] and corpus2=["positive"].
4. For timpa_steer, --steer add calls additive extraction and passes
   steer_mode="add". --steer projection calls projection extraction and passes
   steer_mode="project_out". The examples below use the current run.sh setting,
   --steer add.
5. timpa_hybrid always uses additive steering. Its target prompt detects tokens;
   its target-minus-source vector is used while LLaDA refills those tokens.
6. timpa_steer and timpa_hybrid currently support only IMDb and catdog. The
   corresponding ELIFive and Dolly sections show the calls that are blocked.
